# DCA Tracker 使用说明

这是一款给个人做定投记录和复盘用的网页工具。它适合两种常见场景：

- 你想按固定节奏长期买入，并把每一期的价格、股数、投入金额记录下来。
- 你想按计划管理多只资产的仓位，并随时查看自己离目标路径还有多远。

这个工具支持两种策略：

- `DCA`：每期按固定金额投入，适合想简单执行的人。
- `VA`：根据“当前持仓价值”和“目标价值”的差距来决定本期该投多少，更适合想严格贴着目标路径走的人。

它还支持两种预算方式：

- `固定预算`：你有一笔总预算，准备分成若干期投完。
- `无限定投`：你每期持续投入，没有固定结束时间。

## 先知道这几个页面

底部一共有 4 个页面：

- `总览`：看全局结果，适合复盘和判断现在的仓位状态。
- `本期操作`：录入本期价格、买入股数，并保存本期记录。
- `历史`：查看以前每一期的执行结果，也可以编辑、删除、导出、导入。
- `设置`：新建或修改计划，包括预算、策略、频率和资产配置。

如果你创建了多份计划，页面顶部会出现 `当前计划` 下拉框，可以切换不同计划。

## 第一次使用怎么做

第一次打开时，建议你按下面的顺序来：

### 第 1 步：进入“设置”，创建第一份计划

在 `设置` 页里，你需要先把一份计划配置完整，才能开始记录。

你需要依次填写这些内容：

#### 1. 计划名称

随便起一个自己能看懂的名字即可，例如：

- `2026 美股 VA 定投`
- `长期 ETF 定投`
- `双周成长组合`

#### 2. 预算模式

选择你更接近哪种情况：

- `固定预算`：适合已经准备好一笔钱，想分 6 期、12 期或 24 期慢慢投。
- `无限定投`：适合每月或每两周持续投入工资的一部分，不设结束期。

#### 3. 策略类型

如果你不知道怎么选，可以直接这么理解：

- 想简单、机械执行，选 `DCA定额`
- 想让每期投入跟目标偏差联动，选 `VA定投`

两者区别很直白：

- `DCA` 每期建议投入金额基本固定，只会因为资产权重不同而分配不同金额。
- `VA` 每期会先算“这一期你理论上应该达到多少市值”，再用这个目标减去你当前已经有的持仓价值，得出本期建议投入金额。

#### 4. 预算和周期

如果你选的是 `固定预算`，要填写：

- `总预算（美元）`
- `总期数`
- `保留现金比例`

这里的 `保留现金比例` 可以理解成你不打算立即投入市场、而是预留出来的现金缓冲。  
系统会自动帮你算出：

- `保留现金`
- `可投资金`

如果你选的是 `无限定投`，则不需要填总预算，只需要填：

- `每期计划投入金额（美元）`

这个数值在无限定投模式下更像“参考目标”，不是硬性限制。

#### 5. 执行节奏

在 `定投频率` 里选择：

- `双周`
- `月`

这会影响系统按多少期来计算目标路径。

#### 6. 目标年化收益率（仅 VA 用）

如果你选的是 `VA定投`，还会看到 `目标年化收益率`。

你可以：

- 手动拖动滑杆设置
- 点击 `自动测算`

`自动测算` 会根据你添加的资产 ticker 给出一个大致建议范围，但它只是参考，不代表未来真实收益。

如果你选的是 `DCA`，这一项不需要设置。

#### 7. 添加资产

点击 `添加标的` 后，为每个资产填写：

- `Ticker`：比如 `QQQ`、`VOO`、`TQQQ`
- `显示名称`：给自己看的名字，不写也没关系
- `现有股数`：如果你本来就持有这只资产，可以填当前已有股数；如果从零开始，就填 `0`
- `权重`

注意两点：

- 所有资产的权重加起来必须等于 `100%`
- 只要权重没到 `100%`，系统就不会允许你保存计划

#### 8. 保存计划

右侧的 `保存前检查` 会持续告诉你当前配置是否可以安全保存。  
确认没问题后，点击 `保存计划`。

保存成功后，通常会回到 `总览` 页。

## 创建好计划后，下一步怎么用

真正的日常使用流程，基本就是下面这套：

1. 打开 `本期操作`
2. 录入每个资产本期的价格
3. 确认系统给出的建议金额和建议股数
4. 把你实际买入的股数写进去
5. 选择本期标签并写备注
6. 点击 `确认记录本期操作`
7. 去 `历史` 或 `总览` 看结果

## “本期操作”页怎么填

这一页是你最常用的页面。

### 1. 先看页面顶部

顶部会告诉你：

- 现在是第几期
- 当前策略是什么
- 频率是双周还是月度
- 上一期有没有记录
- 当前有多少个标的
- 已经录入了几个价格

如果你是第一次使用，这里会提示“这是首次执行记录”。

### 2. 给每个资产录入价格

每个资产卡片里都有 `价格输入` 区域。

你可以两种方式输入价格：

- 手动输入
- 点击 `自动获取价格`

自动获取价格依赖服务端代理 Twelve Data。如果 Docker 容器里没有配置 `TWELVE_DATA_KEY`，或者接口失败、超时、额度用尽，系统会提示你手动输入，不会卡住整个流程。

### 3. 看懂每张资产卡片上的 4 个关键数字

每个资产下面都会看到这些信息：

- `当前持仓价值`
- `VA 目标值` 或 `本期固定投入`
- `建议买入金额`
- `建议买入股数`

它们分别是什么意思：

- `当前持仓价值`：你在本期买入之前，这只资产按当前价格计算出来的持仓市值。
- `VA目标值`：只在 VA 策略下出现，表示系统认为你这一期理论上应该达到的目标市值。
- `本期固定投入`：只在 DCA 策略下出现，表示这只资产本期按权重分配后应该投入的固定金额。
- `建议买入金额`：系统根据策略算出来，这一期建议你往这只资产里投多少钱。
- `建议买入股数`：系统把建议金额除以当前价格后，按整数股向下取整/四舍五入得到的建议买入股数。

你可以把它理解成：

- VA 看的是“离目标还差多少”
- DCA 看的是“这期原本就打算投多少”

### 4. 填写“实际买入股数”

这个输入框非常重要。

系统会先自动把 `建议买入股数` 显示在这里，作为默认值。  
也就是说：

- 如果你不改，它会按建议股数保存
- 如果你实际买入和建议不一样，你一定要手动改成真实股数

系统会根据你填的真实股数，自动算出：

- `实际投入金额`

### 5. 选择本期决策标签

页面底部有 3 个标签：

- `正常执行`
- `主动低配`
- `本期暂停`

可以这样理解：

- `正常执行`：基本按计划做了
- `主动低配`：本期故意少买了一些
- `本期暂停`：这一期没有执行买入

这个标签主要是给你后面复盘时看的。

### 6. 写备注

建议每一期都简单写一句，哪怕只写一句也有帮助，例如：

- “纳指回撤较大，按建议执行”
- “本期担心波动，主动低配”
- “已有大额支出，本期暂停”

后面在 `总览` 和 `历史` 里，这些备注都能帮助你回忆当时为什么这么操作。

### 7. 确认保存

确认价格都已经录完后，点击 `确认记录本期操作`。

保存后系统会做两件事：

- 把这一期写入历史记录
- 把各资产的累计持仓股数更新到计划里

## “总览”页怎么看

如果说 `本期操作` 是录入页面，那 `总览` 就是看结果的页面。

### 顶部摘要

顶部会先告诉你这份计划的基础信息：

- `策略`
- `频率`
- `最近记录`
- `计划状态`

这里适合用来快速确认：自己现在看的到底是哪份计划、做到第几期了、最近一次是什么时候执行的。

### 四个核心指标

接下来最值得看的，是这 4 张卡：

#### 1. 当前总市值

表示你当前所有资产按最新录入价格计算出来的总市值。

#### 2. 累计总投入

表示你到目前为止，历史上所有期次实际投入金额的总和。

#### 3. 浮动盈亏

计算方式可以简单理解为：

`当前总市值 - 累计总投入`

如果是正数，就是当前浮盈。  
如果是负数，就是当前浮亏。

#### 4. 剩余可投 / 本期状态

这里会根据模式不同显示不同内容：

- `固定预算` 下显示 `剩余可投`
- `无限定投` 下显示 `本期状态`

如果你是固定预算用户，这里能帮你快速知道预算还剩多少。

### 计划健康度

右侧的 `计划健康度` 适合快速判断执行状态。

#### 固定预算模式下

你会看到：

- `预算推进`
- `可投资金`
- `保留底仓`
- `剩余可投`

这里最重要的是：

- `保留底仓`：你一开始预留不想动的现金
- `可投资金`：真正用于定投推进的那部分钱
- `剩余可投`：当前还没投入掉的可投资金

#### 无限定投模式下

你会看到：

- `累计期数`
- `最新记录`
- `本期状态`

这更像一个长期执行状态面板，而不是预算管理面板。

### 组合轨迹

`组合轨迹` 图里有两条线：

- `实际持仓价值`
- `目标轨迹`

怎么看：

- 如果实际线长期明显低于目标线，说明你的执行偏慢，或者价格变化让你落后于目标路径
- 如果实际线贴近目标线，说明执行比较稳定

这张图尤其适合 VA 用户，因为 VA 本来就是围绕目标路径在执行。

### 仓位结构

`仓位结构` 这块会展示每个资产：

- 当前权重
- 目标权重
- 权重偏离

怎么看：

- `当前权重`：现在这只资产在整个组合里占多少
- `目标权重`：你最初设定它应该占多少
- `权重偏离`：实际仓位和目标仓位差了多少

如果某只资产偏离很大，就说明你的仓位结构已经和最初配置不太一样了。

### 投入节奏

`投入节奏` 图会展示：

- 每一期实际投入多少
- 到当前为止累计投入多少

这张图适合看两件事：

- 你是否在稳定地执行
- 某几期是否突然大幅加仓或明显停顿

### 关键检查点

这个表格非常实用，尤其适合复盘：

- `平均成本`
- `最新价格`
- `权重偏离`

可以这样看：

- `平均成本` 用来判断你当前价格相对成本是贵了还是便宜了
- `最新价格` 下方的百分比，是当前价格相对平均成本的偏离
- `权重偏离` 用来判断仓位是否失衡

如果你只想快速判断组合状态，这个表格和上面的四个核心指标基本已经够用了。

## “历史”页怎么用

`历史` 页是你的执行台账。

### 你可以在这里做什么

- 按标签筛选记录
- 展开查看某一期的详细数据
- 编辑历史记录
- 删除某一期记录
- 导出 CSV
- 导出 JSON 备份
- 导入 JSON 备份

### 展开详情后能看到什么

每一期展开后，你会看到每个资产的：

- 操作价格
- 目标值
- 前持仓价值
- 建议买入金额
- 建议股数
- 实际买入股数
- 实际投入金额

这很适合做复盘，比如看自己到底是按建议执行了，还是偏离了计划。

### 编辑记录适合什么情况

如果你发现之前录错了：

- 价格
- 实际买入股数
- 决策标签
- 备注

可以直接在历史里改。  
保存后，系统会重新计算该计划后续的累计投入和持仓状态。

### 删除记录要注意什么

删除某一期后，系统也会按剩余记录重新计算计划状态。  
这个操作不可撤销，建议谨慎。

### 导出和导入怎么理解

- `导出 CSV`：适合拿去做表格分析
- `导出 JSON`：适合做完整备份
- `导入 JSON`：适合恢复之前导出的备份数据

如果你很在意数据安全，建议养成定期导出 JSON 的习惯。

## 一些很重要的使用提醒

### 1. 数据保存在浏览器本地

如果你用 `npm run dev` 直接打开前端，应用会运行在浏览器 `localStorage` 兼容模式。  
正式部署到 Docker / NAS 后，数据会保存在容器映射出来的本地目录里，并自动生成备份。
这意味着：

- 开发模式下，换浏览器后，原来的数据不会自动跟过去
- Docker 部署后，只要 NAS 上挂载的数据目录还在，迁移容器或换机器都能继续读取
- 最稳妥的做法仍然是定期 `导出 JSON` 备份；Docker 模式下系统还会自动保留最近备份

### 2. 自动价格获取不是必需的

如果你没有配置 API Key，或者接口临时失败，也完全可以继续使用。  
你只要手动填价格即可，不会影响记录功能。

### 3. 权重必须是 100%

只要所有资产权重之和不是 `100%`，系统就不会让你保存计划。  
如果你发现保存按钮点不动，优先检查这个地方。

### 4. “实际买入股数”才是最终记账依据

系统会给你建议值，但最终写入历史的是你确认时的 `实际买入股数`。  
如果你真实下单和建议不一样，请务必把这里改成真实值。

### 5. 清除所有数据前，先做备份

`设置` 页里有 `清除所有数据`。  
这个操作会清掉计划和历史记录，建议先导出 JSON 再执行。

## 如果你完全是第一次上手，最短使用路径

你可以直接照这个最短流程做：

1. 去 `设置`
2. 创建一个计划
3. 添加资产并把总权重调到 `100%`
4. 点击 `保存计划`
5. 去 `本期操作`
6. 给每个资产输入价格
7. 核对建议买入金额和建议股数
8. 把 `实际买入股数` 改成你真实买入的数量
9. 选一个标签，写一句备注
10. 点击 `确认记录本期操作`
11. 去 `总览` 看整体结果
12. 去 `历史` 看本期有没有正确入账

## 一个简单的理解方式

如果你不想一开始就研究所有指标，可以先这样用：

- `设置`：把计划搭好
- `本期操作`：把这一期真实交易录进去
- `历史`：确认数据没错
- `总览`：只看 4 个核心指标和 `关键检查点`

这样已经足够把这个工具用起来了。等你熟悉以后，再去看 `组合轨迹` 和 `仓位结构`，会更容易理解。
